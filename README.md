
# 📖 دليل المطور - مشروع أنيس القلوب (Anis Al-Qulub)

أهلاً بك! هذا الملف هو دليلك الشامل لتشغيل وتطوير تطبيق "أنيس القلوب" على جهازك المحلي. تم كتابة هذا الدليل لضمان تشغيل التطبيق دون أخطاء.

---

## 🛠️ 1. المتطلبات المسبقة (Prerequisites)

قبل البدء، تأكد من تثبيت البرامج التالية على جهازك:

1.  **Node.js** (الإصدار 18 أو أحدث):
    *   هو المحرك المشغل للغة JavaScript خارج المتصفح.
    *   [رابط التحميل (nodejs.org)](https://nodejs.org/) - اختر نسخة **LTS**.
2.  **VS Code** (محرر الأكواد):
    *   أفضل بيئة لكتابة وتعديل الكود.
    *   [رابط التحميل](https://code.visualstudio.com/).

---

## 🚀 2. إنشاء المشروع وتثبيت المكتبات

سنستخدم أداة **Vite** لأنها سريعة جداً وتمنع مشاكل التوافق الشائعة.

### الخطوة الأولى: إنشاء المجلد
افتح "موجه الأوامر" (Terminal) أو PowerShell، واكتب الأوامر التالية بالترتيب:

```bash
# إنشاء مشروع React مع TypeScript
npm create vite@latest anis-app -- --template react-ts

# الدخول لمجلد المشروع
cd anis-app

# تثبيت الحزم الأساسية
npm install
```

### الخطوة الثانية: تثبيت المكتبات الإضافية
يحتاج هذا المشروع لمكتبتين أساسيتين:
1.  `lucide-react`: للأيقونات الجميلة.
2.  `@google/genai`: للربط مع ذكاء Google Gemini.

اكتب هذا الأمر في التيرمينال:
```bash
npm install lucide-react @google/genai
```

---

## 📂 3. هيكلة الملفات (أين تضع الأكواد؟)

هذه أهم خطوة لضمان عمل البرنامج. تأكد أن هيكل المجلدات لديك يطابق هذا الشكل تماماً:

```text
anis-app/
├── index.html          <-- الملف الرئيسي (تعديل العنوان واللغة)
├── package.json
├── .env                <-- ملف المفاتيح السرية (سننشئه)
├── public/             <-- الملفات العامة
│   ├── manifest.json   <-- انسخ محتوى الملف هنا
│   └── sw.js           <-- انسخ محتوى الملف هنا
└── src/
    ├── index.tsx       <-- نقطة البداية (تأكد من ربطه في index.html)
    ├── App.tsx         <-- واجهة التطبيق الرئيسية
    ├── types.ts        <-- تعريفات البيانات
    ├── components/     <-- (أنشئ هذا المجلد)
    │   ├── Header.tsx
    │   ├── EmotionForm.tsx
    │   ├── ResultCard.tsx
    │   ├── SettingsModal.tsx
    │   ├── HistoryModal.tsx
    │   └── Sidebar.tsx
    └── services/       <-- (أنشئ هذا المجلد)
        ├── geminiService.ts
        ├── supabaseService.ts
        └── syncService.ts
```

### تعليمات نقل الملفات:
1.  **المجلد `src/components`**: أنشئه وضع فيه ملفات المكونات (Header, Sidebar, etc...).
2.  **المجلد `src/services`**: أنشئه وضع فيه `geminiService.ts`.
3.  **المجلد `public`**: ضع فيه ملفي `sw.js` و `manifest.json`. هذا مهم جداً لكي يعمل التطبيق بدون إنترنت لاحقاً.
4.  **الملف `index.html`**: تأكد من أن السطر الخاص بالسكريبت يشير للملف الصحيح: `<script type="module" src="/src/index.tsx"></script>`.

---

## 🔑 4. إعداد مفاتيح الربط (API Keys)

لكي يعمل التطبيق بكامل ميزاته، تحتاج للمفاتيح التالية في ملف `.env`:

```env
VITE_API_KEY=AIzaSyxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
# اختيارياً لربط Supabase (النظام الأساسي)
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```
*(استبدل الـ xxx بمفتاحك الحقيقي)*.

---

## ▶️ 5. تشغيل التطبيق

الآن، كل شيء جاهز! لتشغيل البرنامج في وضع المطور:

1.  افتح التيرمينال داخل مجلد المشروع.
2.  اكتب الأمر:
    ```bash
    npm run dev
    ```
3.  سيظهر لك رابط (مثل `http://localhost:5173`). اضغط عليه، ومبروك عليك التطبيق!

---

## 📦 6. بناء النسخة النهائية (للرفع على الإنترنت)

عندما تنتهي من التعديلات وتريد نشر التطبيق:

1.  اكتب الأمر:
    ```bash
    npm run build
    ```
2.  سيظهر مجلد جديد اسمه `dist`. هذا المجلد يحتوي على ملفات الموقع النهائية جاهزة للرفع على أي استضافة مثل Vercel أو Netlify.

---

## ⚠️ حل المشاكل الشائعة

*   **خطأ `Module not found`**: تأكد أنك أنشأت المجلدات (`components`, `services`) ونقلت الملفات بداخلها بشكل صحيح.
*   **خطأ `API Key missing`**: تأكد من إنشاء ملف `.env` وأن اسم المتغير يبدأ بـ `VITE_`. بعد إنشاء الملف، يجب إغلاق التيرمينال وإعادة تشغيله.
*   **الأيقونات لا تظهر**: تأكد من تثبيت مكتبة lucide-react.
